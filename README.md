# rade_v2_voice

A pure-C, BSD-licensed implementation of the `rv_*` DLL interface used by
[RadeCallTest](https://github.com/yi-matsui/RadeCallTest) style Windows applications, built on
[RADE](https://github.com/drowe67/radae) (Radio Autoencoder) V1/V2 and the
[FARGAN](https://arxiv.org/abs/2405.21069) vocoder.

## Background

RadeCallTest originally linked against a `rade_voice.dll` implementation that
is GPL-licensed. `rade_v2_voice.dll` in this repo re-implements the same
19-function `rv_*` interface (verified against the original via
`dumpbin /exports`) from scratch in pure C, so it can be distributed under a
permissive BSD license. It links RADE V2 (primary) and RADE V1 (`src_v1`,
compatibility/interop) against Opus/LPCNet's FARGAN vocoder.

## Layout

| Path | Contents |
| --- | --- |
| `src_v2/` | RADE V2 codec (encoder/decoder/frame-sync/EOO/BPF) and `rade_v2_voice.c`, the `rv_*` DLL entry points |
| `src_v2/tests/` | Standalone C test programs for individual V2 stages (sync, enc, dec, EOO, BPF, loopback, ...) |
| `src_v2/build/` | Windows (`cl.exe`/MSVC) batch build scripts |
| `src_v2/csharp/` | C# `P/Invoke` wrappers and smoke-test harness for consuming the DLL from .NET |
| `src_v2/doc/` | Design notes on the DLL ABI, BPF/pitch limiting, and the V2 spec patch |
| `src_v1/` | RADE V1 pure-C port (encoder/decoder/OFDM/acquisition), used for V1 interop |
| `tools/` | Python scripts that export trained PyTorch weights into the `*_data.c` arrays compiled into `src_v1`/`src_v2` |
| `third_party/opus` | Vendored copy of [xiph/opus](https://github.com/xiph/opus) at commit `940d4e5a`, providing FARGAN/LPCNet (build outputs and downloadable model data excluded) |
| `patches/` | `opus-nnet.h.diff` — applied to `third_party/opus/dnn/nnet.h` on Linux/macOS builds only (not needed for the MSVC static-link build) |
| `scripts/` | `build_opus.bat` — builds `opus.lib` on Windows via CMake/NMake, including FARGAN model data download |

`rade_v2_constants_data.c`, `rade_enc_v2_data.c`, `rade_dec_v2_data.c` (and
their V1 equivalents in `src_v1/`) contain the trained model weights baked in
as C arrays, generated by the scripts in `tools/`. They're large text files
(tens of MB) but are tracked directly in this repo rather than via Git LFS.

## Getting the Opus/LPCNet dependency

FARGAN and the LPCNet feature extraction (`arch.h`, `lpcnet.h`, `freq.h`,
`os_support.h`, `fargan.h`, `cpu_support.h`) come from Xiph's Opus repo. The
full source tree is vendored in this repository under `third_party/opus`, at
commit `940d4e5af64351ca8ba8390df3f555484c567fbb` — the exact commit this
project was built and verified against (the same one
`scripts/build_opus.bat` and the original `BuildOpus.cmake` pin to). No
submodule setup is needed; a plain `git clone` gets everything.

The vendored tree intentionally **excludes the trained model data**
(FARGAN/DRED/OSCE weights: `dnn/*_data.c`, `dnn/*_data.h`,
`dnn/models/*.pth`, `dnn/dred_rdovae_constants.h`). Opus doesn't ship this
data in its own repo either — it must be downloaded once before building:

```bat
cd third_party\opus
dnn\download_model.bat 4ed9445b96698bad25d852e912b41495ddfa30c8dbc8a55f9cde5826ed793453
```

(On Linux/macOS: `./dnn/download_model.sh 4ed9445b...` — same hash.) These
downloaded files are listed in `.gitignore`, so they won't show up as
untracked changes.

### Building opus itself

On Linux/macOS, Opus builds the normal autotools way, with DNN/OSCE and
FARGAN enabled. This path also needs `patches/opus-nnet.h.diff` applied to
`dnn/nnet.h` first (adds a `RADE_EXPORT` visibility macro and generic-layer
function declarations — matches `BuildOpus.cmake`'s `PATCH_COMMAND`):

```bash
patch third_party/opus/dnn/nnet.h patches/opus-nnet.h.diff
cd third_party/opus
./autogen.sh
./configure --enable-osce --enable-dred --disable-shared --disable-doc --disable-extra-programs
make
```

On Windows, build from an **x64 Native Tools Command Prompt for VS**, with
`cmake`, `tar`, and `powershell` on `PATH`. Since the opus source is already
vendored in-tree, run the model download (above) once, then configure and
build directly in `third_party\opus`:

```bat
cd third_party\opus
cmake -B build -G "NMake Makefiles" ^
  -DOPUS_BUILD_SHARED_LIBRARY=OFF -DOPUS_BUILD_TESTING=OFF ^
  -DOPUS_DEEP_PLC=ON -DOPUS_DRED=ON -DOPUS_OSCE=ON
cmake --build build
```

This produces `third_party\opus\build\opus.lib` (git-ignored). Note that
`patches/opus-nnet.h.diff` is **not applied** on this path — it addresses
GCC visibility attributes and isn't needed when statically linking opus into
the DLL under MSVC.

`scripts/build_opus.bat` automates the same steps (same commit, same model
hash, same CMake options), but it was written for a standalone checkout: it
clones `opus` into its working directory if not present. If you prefer the
script, run it in a scratch location, or adjust its clone step to reuse
`third_party/opus`.

## Building rade_v2_voice.dll (Windows / MSVC)

The build scripts under `src_v2/build/` expect these environment variables
(defaults shown, relative to `src_v2/`):

- `OPUS_SRC` — path to the Opus/LPCNet source tree, default
  `..\..\dr-radev2\opus`. Point this at `third_party\opus` from this repo's
  root, e.g. `set OPUS_SRC=..\third_party\opus`
- `OPUS_LIB` — path to the `opus.lib` you built above, default
  `..\..\dr-radev2\opus\build\opus.lib`. e.g.
  `set OPUS_LIB=..\third_party\opus\build\opus.lib`
- `V1_SRC` — path to `src_v1`, default `..\src_v1` (already correct relative
  to `src_v2` in this repo's layout)

From `src_v2/`, with a Visual Studio "Developer Command Prompt" open:

```bat
set OPUS_SRC=..\third_party\opus
set OPUS_LIB=..\third_party\opus\build\opus.lib
build\build_rade_v2_voice.bat
```

This produces `rade_v2_voice.dll`, exporting the `rv_*` functions consumed by
`src_v2/csharp/RadeV2Native.cs`.
