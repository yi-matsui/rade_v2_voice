# rade_v2_voice

A pure-C, BSD-licensed implementation of the `rv_*` DLL interface used by
[RadeCallTest](https://github.com/) style Windows applications, built on
[RADE](https://github.com/drowe67/radae) (Radio Autoencoder) V1/V2 and the
[FARGAN](https://arxiv.org/abs/2405.21069) vocoder.

## Background

RadeCallTest originally linked against a `rade_voice.dll` implementation that
is GPL-licensed. `rade_v2_voice.dll` in this repo re-implements the same
19-function `rv_*` interface (verified against the original via
`dumpbin /exports`) from scratch in pure C, so it can be distributed under a
permissive BSD license. It links RADE V2 (primary) and RADE V1 (`src_v1`,
compatibility/interop) against Opus/LPCNet's FARGAN vocoder.

## Layout

| Path | Contents |
| --- | --- |
| `src_v2/` | RADE V2 codec (encoder/decoder/frame-sync/EOO/BPF) and `rade_v2_voice.c`, the `rv_*` DLL entry points |
| `src_v2/tests/` | Standalone C test programs for individual V2 stages (sync, enc, dec, EOO, BPF, loopback, ...) |
| `src_v2/build/` | Windows (`cl.exe`/MSVC) batch build scripts |
| `src_v2/csharp/` | C# `P/Invoke` wrappers and smoke-test harness for consuming the DLL from .NET |
| `src_v2/doc/` | Design notes on the DLL ABI, BPF/pitch limiting, and the V2 spec patch |
| `src_v1/` | RADE V1 pure-C port (encoder/decoder/OFDM/acquisition), used for V1 interop |
| `tools/` | Python scripts that export trained PyTorch weights into the `*_data.c` arrays compiled into `src_v1`/`src_v2` |
| `third_party/opus` | Git submodule: [xiph/opus](https://github.com/xiph/opus), pinned to the commit providing FARGAN/LPCNet |
| `patches/` | `opus-nnet.h.diff`, applied to the submodule's `dnn/nnet.h` before building |

`rade_v2_constants_data.c`, `rade_enc_v2_data.c`, `rade_dec_v2_data.c` (and
their V1 equivalents in `src_v1/`) contain the trained model weights baked in
as C arrays, generated by the scripts in `tools/`. They're large text files
(tens of MB) but are tracked directly in this repo rather than via Git LFS.

## Getting the Opus/LPCNet dependency

FARGAN and the LPCNet feature extraction (`arch.h`, `lpcnet.h`, `freq.h`,
`os_support.h`, `fargan.h`, `cpu_support.h`) come from Xiph's Opus repo,
included here as a submodule pinned to the exact commit this project was
built and verified against:

```bash
git clone --recurse-submodules <this-repo-url>
# or, if already cloned:
git submodule update --init --recursive
```

This checks out `third_party/opus` at commit
`940d4e5af64351ca8ba8390df3f555484c567fbb`. That commit's `dnn/nnet.h` needs
one small patch (adds a `RADE_EXPORT` visibility macro and generic-layer
function declarations) before building:

```bash
patch third_party/opus/dnn/nnet.h patches/opus-nnet.h.diff
```

### Building opus itself

On Linux/macOS, Opus builds the normal autotools way, with DNN/OSCE and
FARGAN enabled:

```bash
cd third_party/opus
./autogen.sh
./configure --enable-osce --enable-dred --disable-shared --disable-doc --disable-extra-programs
make
```

On Windows/MSVC there's no ready-made project file for the DNN/FARGAN
pieces bundled with Opus, so `opus.lib` needs to be produced separately тАФ
e.g. via a MinGW/MSYS2 toolchain running the same `autogen.sh`/`configure`/
`make` steps above, or by compiling `celt/`, `silk/`, and `dnn/` into a
static library with `cl.exe` yourself. However you build it, you should end
up with a static `opus.lib` (or `libopus.a`) plus the header search paths
below.

## Building rade_v2_voice.dll (Windows / MSVC)

The build scripts under `src_v2/build/` expect these environment variables
(defaults shown, relative to `src_v2/`):

- `OPUS_SRC` тАФ path to the Opus/LPCNet source tree, default
  `..\..\dr-radev2\opus`. Point this at `third_party\opus` from this repo's
  root, e.g. `set OPUS_SRC=..\third_party\opus`
- `OPUS_LIB` тАФ path to the `opus.lib` you built above, default
  `..\..\dr-radev2\opus\build\opus.lib`. e.g.
  `set OPUS_LIB=..\third_party\opus\build\opus.lib`
- `V1_SRC` тАФ path to `src_v1`, default `..\src_v1` (already correct relative
  to `src_v2` in this repo's layout)

From `src_v2/`, with a Visual Studio "Developer Command Prompt" open:

```bat
set OPUS_SRC=..\third_party\opus
set OPUS_LIB=..\third_party\opus\build\opus.lib
build\build_rade_v2_voice.bat
```

This produces `rade_v2_voice.dll`, exporting the `rv_*` functions consumed by
`src_v2/csharp/RadeV2Native.cs`.
